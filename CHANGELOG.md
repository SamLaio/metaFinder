# 變更紀錄

## 2026-07-16

- 改善書名查詢候選排序：當查詢同時含有完整書名與作者時，完整命中書名與作者的候選會優先於只命中部分泛詞的候選，避免例如「新時代，新魔法」被「新時代女性」類同詞書名排在前面。
- 新增回歸測試，覆蓋完整書名與作者命中應優先於高分鬆散詞命中的情境。
- 書名比對會辨識晉江等網路小說頁常見的 `《書名》作者_站名` 標題格式，並將 `jjwxc.net` / `m.jjwxc.net` 納入預設站點搜尋。
- 補上晉江作品頁 URL 規則與「晉江文學城 / 晋江文学城 / jjwxc」查詢提示，讓網路小說 metadata 搜尋更容易收進正確官方作品頁。
- 多詞查詢只命中單一泛詞時不再視為有效候選，避免找不到正確官方頁時把同名詞但不同書、不同作者的書誤報為候選。
- 搜尋候選 JSON 會在候選頂層同步輸出 `cover_url`，並補強封面抽取，支援 `og:image:secure_url`、`twitter:image:src`、JSON-LD `ImageObject.contentUrl` 與常見 lazy-load 圖片屬性。

## 2026-07-13

- 補強 agent 慣例：Calibre 書庫整理流程中，只要 metadata 判斷可泛用，就不應只留在一次性手工判斷，應沉澱到本專案。
- 明確列出適合沉澱的規則類型，包括來源優先序、ISBN/eISBN、作者與譯者清理、出版日、系列、標籤、得獎標籤、不推論來源欄位、站台 parser、封面 URL 與簡轉正詞彙。
- 明確規定沉澱規則時要修改實作或資料檔、補 regression test、更新 README 或使用說明、更新 change log，並執行 `pytest -q`。
- 強化簡轉正自訂替換：OpenCC 轉換前後都會套用 `custom_replacements.tsv`，讓簡體 key 與正體 key 都能命中，例如 `实时` 可正確轉為 `即時`。
